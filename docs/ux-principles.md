# Germany Move Quest - UX Principles

## Vision

Germany Move Quest reduces the stress of relocating to Germany by guiding users through the journey one meaningful step at a time.

The application should feel more like a trusted companion than a task manager.

---

## Design Principles

### Mobile First

The application is designed primarily for mobile devices because relocation tasks are completed in the real world.

Desktop users should experience an optimized layout rather than a stretched mobile interface.

---

### One Primary Action

Each screen should clearly answer:

"What should I do next?"

Avoid presenting multiple competing actions.

---

### Calm, Not Corporate

The interface should use:

- generous whitespace
- subtle shadows
- rounded corners
- minimal borders

The goal is to reduce stress rather than increase cognitive load.

---

### German First

German terminology should appear first.

English explanations should support learning without overwhelming new arrivals.

Example:

Anmeldung
Residence Registration

rather than

Residence Registration (Anmeldung)

---

### Progress, Not Tasks

Users should experience their relocation as a journey.

Progress should be visible and motivating.

The application emphasizes:

- milestones
- achievements
- next recommended step

rather than long task lists.

---

### Helpful Guidance

The application should explain:

- why something matters
- when it should be completed
- what happens next

rather than simply listing tasks.
